Praxis — the natural scientist's harness for Claude

Most "AI for science" makes the model a single oracle you ask for an answer. Praxis instead configures Claude as a small lab that runs the scientific method explicitly — frame a falsifiable question, retrieve real data with provenance, compute, interpret, try to break the result, connect it across fields, and plan what's next. A domain-agnostic core, plus expert packs for gravitational waves and pulsar timing arrays.

Install & prime See it work GitHub

01Core + packs

The method never changes; the domain does. So Praxis splits them. The core runs the scientific method on any field with no pack installed. A pack makes it a fluent expert co-scientist on one field — and ships its knowledge as a provenance-stamped cache under a declared integrity tier, not as prose the model might hallucinate.

praxis core

The loop, the 8 scientist-roles, the integrity rules, and the typed cache engine (pcache · pbuild · pvalidate). Works standalone as a generalist co-scientist.

packs/lvk tier LIVE

Ground-based gravitational waves (LIGO–Virgo–KAGRA / GWOSC) — the flagship Siren pack. 391 GWTC events, asymmetric errors, per-catalog DOIs, live-API ground truth.

packs/pta tier REDERIVE

Pulsar timing arrays (NANOGrav 15yr, EPTA DR2, IPTA DR2 context). No live API — a deterministically re-derivable Zenodo/journal-DOI snapshot, never recalled.

02Install, then prime

Install the core and one or more packs, then run /init once — a few minutes, mostly local reads — to cache an expert briefing into Claude's persistent memory.

claude — install
# in Claude Code
/plugin marketplace add https://github.com/<you>/praxis
/plugin install praxis@praxis        # domain-agnostic core
/plugin install praxis-lvk@praxis    # gravitational-wave pack (Siren)

/init lvk   # one-time: reads the plugin + pack, writes an expert
         # briefing into memory. Future sessions start fluent.
  1. Resolve the pack — argument, sole-pack auto-detect, or ask. Zero packs → runs as a generalist.
  2. Gather the briefing — lib/memory_prime.py reshapes the method loop + the pack's conventions / scope-traps / data-paths / landmark DOIs (deterministic, zero tokens).
  3. Write to memory — praxis-method.md + praxis-domain-<id>.md land in Claude's memory dir and are indexed in MEMORY.md.
  4. Activate — the pack id is written to .praxis/active; a .praxis/primed marker is stamped.

Core works without /init. Priming just makes it an expert: afterward the briefing is recalled as background context every session, so the loop runs fluently without re-reading the plugin.

/investigate Is GW190814's secondary a neutron star or a black hole?

domain-expert-lvk → conventions + the mass-gap ambiguity backstop

next-steps     → what a tidal measurement / EM counterpart would resolve
# honest conclusion, licensed by open_limits: the source class is unresolved.

04Integrity, with teeth — and honest about its limits

Cache integrity is mechanically enforced, not merely documented. pvalidate.py runs 11 contract gates per pack; pcache.py controls the output bytes so it physically cannot emit a bare number or an unprovenanced value.

python lib/pvalidate.py packs/lvk packs/pta
  OK   gate 3: completeness: len(entities)=391 == results_count_reported=391
  OK   gate 4: DOIs well-formed/resolvable       # empty-string rejected; null+flag ok
  OK   gate 7: record_params ⊆ cache params
  OK   gate 9: no unfilled expert-judgment slots
  => lvk: GREEN
  WARN gate 5: transcription_audit present (tier REDERIVE)
  => pta: GREEN-UNVERIFIED   # honest: re-derivable, but no human audit yet
Oracle tierWhat grades itIntegrity guarantee
LIVEIndependent live-API diff at grade time (GWOSC v2)Full mechanical cache integrity + live-fallback safety net. packs/lvk.
REDERIVEDeterministic rebuild must byte-matchRe-derivable integrity; no live oracle; community-unverified until a human audit. packs/pta.
ASSERTEDLLM-transcribed; no machine oracleMethod + discipline, not the mechanical guarantee — every value banners UNVERIFIED-CACHE.

And the honest boundary: Tier-1 deep-loop generation is eval-gated, not categorically guaranteed (the tuned GW pack measured ~89% provenance). "Mechanically enforced" is true for the cache and Tier-0 serving; it is a measured rate, not a proof, for free generation. Full detail in ARCHITECTURE.md.

05The lab (roles → agents)

The principal investigator does not produce the final claim — it frames, delegates, and holds the work to the standard.

principal-investigator

Frames the question, routes the loop, keeps the record.

literature-reader

Reads sources; extracts claim / method / assumptions / limits; cites.

domain-expert-<id>

Field physics, conventions, and the scope/significance backstop — from the active pack.

analyst-coder

Reproducible analysis on the pack's clients + the SBI stack; pins seeds + provenance.

skeptic

The mandatory adversarial pass: priors, systematics, selection, reproducibility.

interpreter

Separates measured / inferred / speculative — never lets inference wear a measurement's clothes.

cross-domain-bridge

Maps a result into what it constrains in adjacent fields.

teacher

Audience-tuned explanation; honesty about confidence over tidiness.

next-steps

Analysis-side and observation-side gap map; essential vs possible.
